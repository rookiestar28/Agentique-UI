# UI Lite Closeout Checklist

Status: UI Lite repository baseline

This checklist defines what the UI Lite baseline could claim and what remained evidence-gated before later supported-local-only runner work and any desktop release claim.

## Current Scope

The UI Lite baseline provides:

- React/Vite app shell with local workspace navigation.
- Untrusted import-intent validation.
- Scoped package verification with digest and cleanup failure behavior.
- Local library index abstraction for verified resource records.
- Safe static previews for text, structured data, metadata, source, logs, diffs, and graph summaries.
- Non-executing handoff descriptors for user-reviewed export or agent-client import paths.

The UI Lite baseline does not provide:

- Released desktop installer.
- Signed update channel.
- Supported-local-only runner evidence.
- Sidecar adapter execution.
- Local secrets vault.
- Automatic arbitrary-resource execution.

## Accessibility And Keyboard Baseline

The UI Lite shell uses native controls and labelled regions:

- Primary navigation uses `button` elements with visible text labels.
- The main workspace and active page region use accessible labels.
- Panels use heading labels through `aria-labelledby`.
- The import textarea has a visible label linked through `htmlFor`.
- Status, preview, and handoff output are rendered as text, lists, and tables.
- No untrusted HTML is injected into the DOM.

Acceptance evidence: `npm run validate` must pass after UI changes.

## Installer, Update, And Rollback Gate

No installer, updater, or release artifact is generated by UI Lite.

Before any installable desktop claim, future evidence must cover:

- Windows, macOS, and Linux build targets.
- Code signing and key custody.
- Signed update metadata and artifact verification.
- Installer and uninstaller smoke checks.
- Update, downgrade, and rollback behavior.
- Release artifact provenance and digest records.
- Emergency revocation and recovery procedure.

## Desktop Runtime Safety Mapping

The UI Lite baseline remains non-executing:

- Tauri package bundling is disabled for the current baseline.
- Default Tauri capability grants no runtime permissions.
- Resource bytes are not imported after failed verification.
- Static preview escapes or metadata-renders untrusted content.
- Handoff descriptors do not execute commands or start bridges.
- Public-boundary validation rejects private planning markers, local absolute paths, and secret-like values.

## Required Validation

Run:

```powershell
npm run validate
git status --short --branch
```

The validation gate includes schema checks, Tauri config checks, closeout assertions, typecheck, production build, unit tests, and public-boundary scan.

## Non-Claims

UI Lite is not a released desktop product, installer, updater, production desktop runtime, generic sidecar runtime, sandboxed execution engine, hosted runtime, or universal workflow runner.

Later Agentique UI validation adds supported-local-only Graph/Run evidence under separate gates. That later evidence does not change the UI Lite baseline into a release, installer, updater, hosted runtime, universal runtime, or automatic arbitrary-resource execution claim.
